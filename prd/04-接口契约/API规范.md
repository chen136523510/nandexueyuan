# API 规范

## 统一响应格式

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | 0=成功，非0=失败 |
| message | string | 提示文案 |
| data | object/array/null | 业务数据 |

## 错误码约定

| code | HTTP Status | 含义 |
|------|-------------|------|
| 0 | 200 | 成功 |
| 1001 | 400 | 参数错误 |
| 1002 | 401 | 未登录 |
| 1003 | 403 | 无权限 |
| 1004 | 404 | 资源不存在 |
| 1005 | 409 | 状态冲突（如重复操作） |
| 5000 | 500 | 服务器内部错误 |

## 鉴权

- 登录成功返回 JWT，存于 localStorage
- 请求头：`Authorization: Bearer <token>`
- axios 拦截器自动注入

## 现有接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/hello` | 联通性测试，返回 `{ message: string }` |
