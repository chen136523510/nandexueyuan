# 技术选型

## 整体架构

```
用户浏览器
    ↓ HTTPS
Nginx（宝塔自带，反向代理 + 静态托管）
    ↓ /api/* 转发
Node.js 后端（端口 3000，PM2 守护）
    ↓
SQLite 数据库
```

## 技术栈

| 层 | 技术 | 版本 | 理由 |
|---|------|------|------|
| **前端框架** | Vue 3 | ^3.5.13 | 主流、组合式 API、生态成熟 |
| **构建工具** | Vite | ^6.0.5 | 构建快、HMR、Vue 官方推荐 |
| **前端路由** | Vue Router | ^5.1.0 | Vue 官方 |
| **前端状态** | Pinia | ^3.0.4 | Vue 3 官方推荐 |
| **HTTP 客户端** | Axios | ^1.18.1 | 标配，拦截器机制 |
| **后端框架** | Express | ^4.21.0 | 教程多、生态成熟 |
| **数据库** | SQLite | 待接入 | 20人使用，零运维，几乎不占内存 |
| **ORM** | Prisma | 待接入 | 类型安全、迁移方便 |
| **认证** | JWT + bcrypt | 待接入 | 标准方案 |
| **进程管理** | PM2 | 待接入 | 守护 Node 进程，开机自启 |
| **反向代理** | Nginx（宝塔自带） | — | 转发 + SSL 证书 |

## 内存预算（1.6 GiB 内可承受）

| 组件 | 占用 |
|------|------|
| 系统 + 宝塔 | ~400 MB |
| Nginx | ~50 MB |
| Node.js 后端 | ~80-150 MB |
| SQLite | ~10 MB |
| **总计** | **~600 MB**，剩余 ~1 GB 余量充足 |

## 已排除方案

- **Next.js**：构建峰值需 1.5-3 GB 内存，1.6 GiB 服务器无法本地构建
- **Java/Spring Boot**：内存吃太多
- **Python/Django**：Vue 已选 JS 栈，统一用 Node 更简单
- **PHP**：偏老，学习价值不如 Node.js
