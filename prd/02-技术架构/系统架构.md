# 系统架构

## 前端目录结构（Vue 3 + Router + Pinia + Axios）

```
src/
├── api/              # 请求层
│   ├── index.js      # axios 实例 + 请求/响应拦截器
│   └── hello.js      # 接口模块（按业务拆分）
├── components/       # 通用组件
│   └── AppFooter.vue
├── router/           # 路由层
│   └── index.js      # 路由配置
├── stores/           # 状态管理层（Pinia）
│   └── hello.js      # store 模块
├── styles/           # 样式层
│   ├── variables.css # 设计令牌（CSS 变量）
│   └── base.css      # 全局基础重置
├── views/            # 页面视图层
│   ├── HomeView.vue  # 首页
│   └── AboutView.vue # 关于页
├── App.vue           # 根组件（仅 router-view）
└── main.js           # 入口（挂载 pinia + router）
```

### 分层职责

| 层 | 职责 | 约束 |
|----|------|------|
| `api/` | 封装 HTTP 请求，对接后端接口 | 不含业务逻辑，只做请求/响应映射 |
| `router/` | 路由定义与守卫 | 不含数据请求 |
| `stores/` | 全局状态管理 | 通过 api/ 层获取数据，不直接 fetch |
| `views/` | 页面级组件 | 组合 components/ 与 stores/ |
| `components/` | 可复用 UI 组件 | 不直接访问 store，通过 props/emits 通信 |
| `styles/` | 全局样式与设计令牌 | 组件内样式用 scoped |

### 数据流

```
View ──调用──▶ Store ──调用──▶ API ──HTTP──▶ 后端
View ◀─响应── Store ◀─响应── API ◀─JSON─── 后端
```

## 后端目录结构（Express 分层）

```
server/src/
├── routes/           # 路由层
│   └── api.js        # API 路由聚合
├── controllers/      # 控制器层
│   └── helloController.js
├── middleware/       # 中间件层
│   └── errorHandler.js
└── index.js          # 入口（挂载路由 + 中间件）
```

### 分层职责

| 层 | 职责 |
|----|------|
| `routes/` | 路由定义，将 URL 映射到 controller |
| `controllers/` | 接收请求，调用业务逻辑，返回响应 |
| `middleware/` | 横切关注点：错误处理、鉴权、日志 |
| `index.js` | 应用入口，挂载全局中间件与路由 |

### 请求处理流

```
Request → cors → jsonParser → router → controller → response
                                          ↓
                                     errorHandler（异常时）
```

## 开发环境

| 服务 | 地址 | 启动命令 |
|------|------|---------|
| 前端 Dev | http://localhost:5173 | `node node_modules/vite/bin/vite.js` |
| 后端 API | http://localhost:3000 | `cd server; node src/index.js` |
| API 代理 | `/api/*` → localhost:3000 | vite.config.js 已配置 |

> 注：npm 11 存在 semver bug，当前用 pnpm standalone bundle 替代。
