## 需求详情

### R-004 版本公告系统

> 登记日期：2026-07-20 | 登记人：陈梓键

#### 背景

当前首页公告栏为单条文本、纯人工维护，无法体现版本迭代历程。每次发版后用户不知道改了什么、未来要做什么。需要将版本信息同步到公告系统，让用户及时了解更新内容。

#### 需求概述

在现有首页公告栏基础上，扩展为**版本公告系统**：
- 公告栏仍展示当前最新版本摘要
- 新增「版本历史」按钮（全员可见），点击查看完整版本历史
- 公告内容由后端数据库管理，发版时由 AI 从 changelog 提取并生成公告文本写入

#### 功能点

| # | 功能 | 说明 |
|:-:|------|------|
| 1 | 版本管理 | 每次发版生成一条版本记录，含版本号、日期、更新内容、未来规划 |
| 2 | 版本号规则 | 语义化 + 日期：`v1.1.0 (2026-07-20)` |
| 3 | 公告内容分类 | 每条版本记录包含两部分：**本次更新**（新增/优化/Bug修复）+ **未来规划**（下一步计划） |
| 4 | 首页公告栏 | 展示最新版本摘要（如"v1.1.0 已发布：新增德塔NPC精灵、优化男德通检索逻辑"） |
| 5 | 版本历史按钮 | 公告栏旁加按钮，点击弹出/跳转完整版本历史列表 |
| 6 | 后端 API | 新增 `GET /api/announcement/versions` 返回版本列表；现有 `GET /api/announcement` 返回最新版本摘要 |
| 7 | 数据库 | 新增 `Version` 表（或扩展现有 Announcement 表），字段：版本号、日期、更新内容（JSON）、未来规划（JSON） |
| 8 | 发版流程 | AI 从 changelog.md 提取本次变更 -> 优化措辞为公告文本 -> 写入数据库 |

#### 公告文本风格

- 用语准确、无歧义，不引起争议
- 不需要特别官方，可以适度轻松
- 示例：

> **v1.1.0 (2026-07-20)**
>
> 📋 本次更新：
> 1. 新增了德塔 NPC 精灵及立绘
> 2. 优化了男德学院男德通模块的检索逻辑
>
> 🚀 未来规划：
> 1. 预期在德塔接入怪物系统和战斗系统
> 2. 男德学院新增导师墙

#### 现有系统衔接

- 现有 `Announcement` 表（id:1 单条记录）需迁移或扩展
- 现有 `GET/PUT /api/announcement` 接口保留兼容
- 管理后台 `MainView.vue` 的公告编辑功能需同步升级
