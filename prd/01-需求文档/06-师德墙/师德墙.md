# 师德墙 PRD

## 1. Why-Who-What

- **业务背景**：男德学院是约 20 人的朋友圈限定社区。现有功能偏工具向（聊天检索、游戏世界），缺少用户之间的轻社交互动。需要一个类似校园墙 / 朋友圈 / 贴吧的模块，让用户分享图文动态、互动交流，增强社区氛围
- **目标用户**：已注册并登录的全体成员（含 admin / super_admin）
- **功能范围**：发布图文动态、全站可见、评论、点赞、删除

## 2. 角色体系

| 角色 | 标识 | 权限 |
|------|------|------|
| 院长 | `super_admin` | 发帖/评论/点赞 + 删除任意动态和评论 |
| 管理员 | `admin` | 发帖/评论/点赞 + 删除任意动态和评论 |
| 成员 | `member` | 发帖/评论/点赞 + 删除自己的动态和评论 |

### 权限矩阵

| 操作 | 院长 | 管理员 | 成员 |
|------|:----:|:------:|:----:|
| 浏览动态列表 | ✓ | ✓ | ✓ |
| 发布动态 | ✓ | ✓ | ✓ |
| 评论/点赞 | ✓ | ✓ | ✓ |
| 删除自己的动态/评论 | ✓ | ✓ | ✓ |
| 删除他人的动态/评论 | ✓ | ✓ | ✗ |

## 3. 数据模型

### Post（动态）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Int | PK, 自增 | |
| authorId | Int | FK -> User.id, 级联删除 | 作者 |
| content | String | 非空，最长 500 字 | 文字内容 |
| image | String? | 可空 | 图片路径（/uploads/wall/xxx.jpg） |
| createdAt | DateTime | 默认 now | 创建时间 |
| updatedAt | DateTime | 自动更新 | 更新时间 |

### Comment（评论）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Int | PK, 自增 | |
| postId | Int | FK -> Post.id, 级联删除 | 所属动态 |
| authorId | Int | FK -> User.id, 级联删除 | 评论者 |
| content | String | 非空，最长 500 字 | 评论内容 |
| createdAt | DateTime | 默认 now | 创建时间 |

### Like（点赞）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Int | PK, 自增 | |
| postId | Int | FK -> Post.id, 级联删除 | 所属动态 |
| userId | Int | FK -> User.id, 级联删除 | 点赞者 |
| createdAt | DateTime | 默认 now | 创建时间 |

> 唯一约束：`[postId, userId]`，一人一帖一赞，重复点赞为 upsert 无副作用。

## 4. 功能详述

### 4.1 浏览动态

- **入口**：导航栏「师德墙」，位于「男德通」与「男通讯录」之间
- **可见范围**：所有登录用户可见全部动态（朋友圈限定，无公开匿名）
- **排序**：时间倒序（最新在上）
- **布局**：横向卡片排列，从左到右滚动，类似画展浏览体验
- **展示信息**：作者头像（昵称首字）、昵称、发布时间、文字内容、图片、点赞数、评论数

### 4.2 发布动态

- **触发**：点击「+ 发动态」按钮展开表单
- **内容**：文字（最长 500 字）+ 图片（1 张，可选）
- **校验**：文字和图片至少有一项
- **图片限制**：jpg/jpeg/png/webp/gif，单张 5MB 以内
- **图片存储**：本地磁盘 `server/uploads/wall/`，文件名 `wall_{时间戳}_{随机串}.{ext}`
- **字数计数**：实时显示 `当前字数/500`
- **发布后**：动态插入列表最前方，表单清空收起

### 4.3 评论

- **触发**：点击动态的评论按钮，展开/收起评论区
- **内容**：纯文字，最长 500 字
- **展示**：一级评论（不支持二级回复），按时间正序排列
- **输入**：评论输入框 + 发送按钮，支持回车发送
- **评论数**：动态卡片上实时显示评论总数

### 4.4 点赞

- **触发**：点击动态的爱心按钮
- **行为**：toggle 切换（已赞 -> 取消，未赞 -> 点赞）
- **视觉**：未赞 🤍，已赞 ❤️（莫兰迪红色）
- **点赞数**：实时更新

### 4.5 删除

- **动态**：作者本人或管理员可删除，删除时同步删除关联图片文件、评论、点赞
- **评论**：评论者本人或管理员可删除
- **确认**：删除前弹出 confirm 确认框

## 5. 种子数据

为避免用户首次进入看到空白页面，系统预置 3 条默认动态：

| # | 作者 | 内容 | 图片来源 |
|:-:|------|------|---------|
| 1 | 陈梓键（院长） | 想象力比知识更重要。知识是有限的，而想象力概括着世界上的一切，推动着进步，并且是知识进化的源泉。 | Wikimedia Commons 公共领域（爱因斯坦肖像） |
| 2 | 陈梓键（院长） | 如果我看得更远，那是因为我站在巨人的肩膀上。 | Wikimedia Commons 公共领域（牛顿画像） |
| 3 | 丘序明 | 低迷。 | 用户提供 |

- 种子图片压缩至 800×800，存放 `server/uploads/wall-seed/`
- 种子脚本：`server/prisma/seedWall.js`，幂等执行（已有数据时跳过）

## 6. 接口契约

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|:----:|------|
| GET | `/api/wall/posts` | ✓ | 获取动态列表（倒序，含作者/评论/点赞） |
| POST | `/api/wall/posts` | ✓ | 发布动态（multipart/form-data：content + image） |
| DELETE | `/api/wall/posts/:id` | ✓ | 删除动态（作者或管理员） |
| POST | `/api/wall/posts/:id/comments` | ✓ | 发表评论 |
| DELETE | `/api/wall/comments/:id` | ✓ | 删除评论（作者或管理员） |
| POST | `/api/wall/posts/:id/like` | ✓ | 点赞 |
| DELETE | `/api/wall/posts/:id/like` | ✓ | 取消点赞 |
