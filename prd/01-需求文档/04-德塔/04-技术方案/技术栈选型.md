# 德塔（NDO）技术栈选型

> 版本：V1 | 日期：2026-07-13 | 状态：已确认

---

## 1. 选型决策

| 层 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| 渲染引擎 | Phaser | 3（稳定版） | 2D 侧视角原生支持，内置 Tilemap / Arcade 物理 / 动画 / 音频，社区生态最活跃 |
| 多人同步 | Colyseus | 0.15+ | 权威服务器，开箱即用：房间管理、状态同步（delta encoding + binary Schema）、防作弊 |
| 地图编辑 | Tiled Map Editor | 最新 | 可视化编辑 2D 瓦片地图，导出 JSON，Phaser 原生解析 |
| 前端框架 | Vue 3（现有） | 不变 | 德塔入口是 Vue 路由的一个页面，Phaser Canvas 挂载在 Vue 组件内 |
| 后端 API | Express（现有） | 不变 | 现有 :3000 端口，AI 助手 / 公告等功能不动 |
| 游戏后端 | Colyseus Server | 0.15+ | 独立进程 :2567，PM2 管理，不干扰 Express |
| 物理引擎 | Arcade Physics | Phaser 内置 | MVP 够用（重力 + AABB 碰撞），后续引入战斗可切 Matter.js |

---

## 2. 视角决策：侧视角（泰拉瑞亚风格）

曾考虑俯视角（宝可梦 / 星露谷 / Gather.town），最终选择**侧视角**。

| 对比维度 | 俯视角 | **侧视角（已选）** |
|----------|--------|-------------------|
| 移动 | 四方向 | 左右 + 跳跃 |
| NPC 交互 | 自然 | 走过去即可 |
| 后续战斗 | 受限 | **原生支持** |
| 后续挖掘/放置 | 受限 | **原生支持** |
| 后续建造 | 受限 | **原生支持** |

**核心原因**：用户目标是类 MC（我的世界）的 2D 版本，侧视角是 2D 版 MC 的最佳呈现方式，不会限制后续战斗/挖掘/建造的拓展。

## 3. 美术风格决策：SNES 32x32 精细像素风

| 维度 | 设定 |
|------|------|
| 风格 | SNES 16-bit（星之卡比超豪华版、圣剑传说 3 级别） |
| 瓦片 | 32x32 px |
| 角色 | 32x64 px，四方向行走动画 |
| 立绘 | 512x512 px，交互时弹出 |
| 色彩 | 每精灵 ≤16 色 |
| 生成方式 | 黑机 ComfyUI（SDXL + Pixel-Art-XL LoRA） |

详见 [02-设计/美术设计规范.md](../02-设计/美术设计规范.md)。

## 4. 服务器承载评估

| 服务器规格 | 2核 2G 40G 系统盘（阿里云 ECS） |
|------------|--------------------------------|
| 目标并发 | **≤30 人**（团队内部使用） |
| MVP 内存峰值 | ~800MB（Express + Colyseus + Nginx） |
| 后期（战斗+物理） | ~1.1GB |
| 结论 | **毫无压力，30 人量级属于大炮打蚊子** |

> 性能优化（视野裁剪、delta encoding、Matter.js 休眠）在 MVP 阶段不需要，后续按需引入。
