## 1. 核心原则：色块先行，美术后替换

美术资源未就位时，使用**色块占位**先跑通所有逻辑。美术就绪后只改一行加载路径即可。

### 1.1 占位对照表

| 真实资源 | 占位方式 | 后续替换 |
|----------|----------|----------|
| 草地瓦片 | 绿色 32x32 矩形 | 改 `this.load.image` 的 key |
| 泥土瓦片 | 棕色 32x32 矩形 | 同上 |
| 墙壁瓦片 | 灰色 32x32 矩形 | 同上 |
| 玩家角色 | 蓝色 32x64 矩形 | 改 `this.load.spritesheet` 的 key |
| NPC 角色 | 黄色 32x64 矩形 | 同上 |
| 可交互物品 | 红色 32x32 矩形 | 同上 |
| NPC 立绘 | 纯色矩形 + 文字 | 替换为 512x512 PNG |
| 地图 | 代码生成 100x30 网格 | 替换为 Tiled JSON |
| Logo | Δ 符号文字 | 替换为像素 PNG |

### 1.2 占位实现方式

使用 Phaser 的 `this.make.graphics()` 生成色块纹理，**不需要外部图片文件**：

```javascript
// PreloadScene.js 占位纹理生成
const gfx = this.make.graphics()
gfx.fillStyle(0x4CAF50)   // 绿色草地
gfx.fillRect(0, 0, 32, 32)
gfx.generateTexture('tile_grass', 32, 32)

gfx.fillStyle(0x2196F3)   // 蓝色玩家
gfx.fillRect(0, 0, 32, 64)
gfx.generateTexture('player_default', 32, 64)
```

### 1.3 替换流程

美术资源就绪后，在 `PreloadScene.js` 中将占位生成替换为真实加载，**key 保持不变**：

```javascript
// 替换前（占位）
// this.make.graphics()...generateTexture('tile_grass', 32, 32)

// 替换后（真实资源）
this.load.image('tile_grass', '/game/tilesets/grass.png')
```

## 2. 开发阶段

| 任务 | 文件 | 说明 |

| Phaser 配置 | `game/config.js` | 已创建，800x600、像素风、arcade 物理、重力 |

| 启动场景 | `game/scenes/BootScene.js` | 初始化，跳转 Preload |
| 预加载场景 | `game/scenes/PreloadScene.js` | 生成占位纹理，显示进度条 |
| 主世界场景 | `game/scenes/WorldScene.js` | 渲染地图、创建角色、物理碰撞 |
| 玩家对象 | `game/objects/Player.js` | 移动（WASD）、跳跃、方向翻转、动画状态 |

**验收标准**：浏览器打开 `/nde`，看到色块地图（三层塔楼 + 外部绿化 + 云顶），蓝块角色可用 WASD 移动 + 空格跳跃，底部面板（角色信息/聊天/小地图），按 E 与 NPC/物品/大门交互，按 Enter 聊天。

## 3. 依赖安装清单

### 3.1 前端（根目录）

```bash
pnpm add phaser colyseus.js
```

### 3.2 游戏后端（game-server/）

```bash
cd game-server
pnpm install --ignore-workspace
```

依赖已在 `game-server/package.json` 中定义：
- `colyseus` ^0.15.0
- `@colyseus/ws-transport` ^0.15.0
