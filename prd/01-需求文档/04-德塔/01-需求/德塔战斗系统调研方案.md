> 决策依据：社交驱动 + 纯PVE + 20人小圈子 + 世界观先行 + 线上服务器 2核2G 性能约束

## 0. 核心设计哲学（已确认）

| 前提 | 决策 | 理由 |
|------|------|------|
| 留存动机 | **社交驱动** | 20 人熟人圈，留存靠共同回忆和人情味，不靠排行榜和数值碾压 |
| 战斗模式 | **纯 PVE** | 成员之间不互相攻击。熟人 PK 容易产生矛盾，聚焦合作打怪 |
| 设计尺度 | **小圈子优先** | 不套用 MMO 的副本/公会/拍卖行，设计"小圈子才好玩"的机制 |
| 世界观先行 | **世界观驱动游戏性** | 世界观是战斗/装备/怪物的**前置任务**，不是附属文档。游戏性从世界观中提炼，不是反过来设计机制再套世界观 |

## 1. 世界脉络（世界观骨架）

> 本节是黑机世界观创作的输入骨架。黑机基于此骨架用酒馆+DeepSeek 创作叙事层，白机从叙事层提炼游戏性。
>
> 分工：酒馆（叙事创作）→ 我（游戏性提炼）→ 代码实现

### 1.1 世界地图总览

**"德塔"一词有两层含义**：(1) 开发代号——我们开发的这一游戏模块；(2) 世界观中的实体——男德学院在异世界建立的第一座根据地。

德塔及塔外区域合起来构成**第一大块区域"德塔"**，是男德学院在此空间开辟的第一块落脚点。

### 1.2 区域设计

| 区域 | 解锁方式 | 定位 | 核心玩法 |
|------|---------|------|---------|
| **德塔塔楼（三层）** | 全部开放 | 安全区/社交中心/复活点 | 社交聊天、教学引导、BOSS 挑战 |
| **近塔森林** | 开放 | 新手战斗区 | 低级怪物、裂隙封印练习 |
| **裂缝外围** | 自由解锁（累计参与 3 次 BOSS） | 中级战斗区 | 精英怪物、更多裂隙 |
| **异世界腹地** | 自由解锁（累计参与 10 次 BOSS） | 高级战斗区 | 强力怪物、稀有材料 |
| **深渊裂隙** | 事件解锁（全社区累积材料达标） | 隐藏区域 | 最高难度、独有装备配方 |

**混合解锁逻辑**：塔楼线性递进（一层探索→解锁二层→二层探索→解锁三层）+ 塔外区域自由解锁（达到贡献值即可进入，不强制顺序）。

### 1.3 世界观核心设定（待黑机酒馆创作填充）

以下设定是黑机世界观创作的**输入骨架**，黑机基于此用酒馆+DeepSeek 填充叙事层：

**核心事件链**（世界观主线脉络，不是玩家游戏主线）：
1. 异次元位面与原位面接壤 → 空间裂隙出现
2. 异次元生物（虚空生物）从裂隙涌入 → 与原住民冲突
3. 男德学院发现裂缝 → 在传送门附近建立德塔作为落脚点
4. 肃清塔周围森林的怪物 → 但裂隙无法完全封印
5. 玩家进入德塔 → 走出森林 → 发现裂隙 → 击杀怪物 → 尝试封印

**关键设定待黑机创作**：
- 异次元位面的名称、性质、规则
- 空间裂隙的形成机制（为什么难以完全封印？）
- 虚空生物的生态（为什么入侵？有没有智慧个体？）
- 原住民的历史和现状（他们知道裂缝吗？在抗争吗？）
- 男德学院为什么选择这片区域建塔？

### 1.4 从世界观到游戏性的提炼方法

这是核心方法论。**不是先定机制再套世界观，而是从世界观设定中自然推导出玩法**。

**示范：空间裂隙 → 完整游戏循环**

世界观设定说"空间裂隙时不时出现怪物，无法完全封印" → 这自然推导出：

```
裂隙机制（世界观设定）
    │
    ├─ 裂隙在森林中随机出现 → 定时刷新怪物（游戏性：有规律的怪物刷新点）
    │
    ├─ 裂隙持续泄漏怪物 → 怪物会越来越多（游戏性：不清理就变难）
    │
    ├─ 玩家击败怪物后可以封印裂隙 → 封印后该区域暂时安全（游戏性：目标感+成就感）
    │
    ├─ 怪物掉落粘液/骨骼等材料 → 用于装备系统（游戏性：资源收集）
    │
    └─ 精英怪物带成品装备 → 直接获取（游戏性：高风险高回报）
```

**这个提炼链条是世界观→游戏性的典型路径**。其他设定（NPC 故事、区域生态、怪物弱点）都应该走同样的提炼路径。

**提炼规则**：
1. 世界观中的"问题" → 游戏中的"目标"（裂隙→封印）
2. 世界观中的"限制" → 游戏中的"规则"（无法完全封印→定时刷新）
3. 世界观中的"资源" → 游戏中的"收集品"（怪物材料→装备合成）
4. 世界观中的"人物" → 游戏中的"NPC"（原住民→对话/任务线索）

### 1.5 世界脉络记录

以下脉络是世界观创作的**方向线索**，不是强制叙事线。黑机创作时参考此脉络，确保叙事与游戏性方向一致：

**世界脉络层 1：环境叙事**（从世界设定本身讲故事）
- 空间裂隙的存在 → 这个世界"不正常"，玩家感受到危机感
- 塔楼是人工据点 → 与荒野形成对比，安全区与危险区的边界感
- 森林的"被肃清"痕迹 → 说明这里发生过战争，暗示历史
- 每个区域的生态差异 → 裂缝越深，虚空生物越密集，世界越"不正常"

**世界脉络层 2：人物故事**（从 NPC 对话发现故事）
- 男德通：为什么被部署在德塔？她有没有关于裂隙的秘密？
- 院长：建造德塔的决策过程，是否有人反对？
- 野外 NPC：异世界原住民的生存状态，他们是逃离还是留守？
- 关键：没有感叹号，没有任务列表，没有"必须做 X 才能推进"

**世界脉络层 3：探索推进**（从区域解锁发现更多）
- 每解锁一个新区域 → 发现更多关于裂缝的线索
- 每封印一个裂隙 → 获得一块"历史碎片"
- 累积碎片 → 解锁关于裂隙真相的更多信息
- 真相不是一次性揭示的，而是随着探索逐步拼凑

### 1.6 我（白机）vs 酒馆（黑机）的分工

> **硬件澄清**：白机 = 荣耀笔记本（工作日白天在公司用）；黑机 = 本地主力机（R7 9700X + RTX 4070 + DeepSeek，晚上用）。**2核2G 是云端服务器**（线上环境）的性能约束，不是白机或黑机本身的限制。

| 维度 | 我（白机，荣耀笔记本 + ZCode） | 酒馆（黑机，本地 R7 9700X+DeepSeek） |
|------|--------------------------|-------------------------------------|
| **强项** | 结构化输出（表格/关系图/数据模型/MECE）、从世界观提炼游戏性、写代码做技术方案 | 沉浸式叙事创作（角色扮演/多轮迭代）、保持设定一致性（世界书）、批量产出对话/小传 |
| **世界观创作** | 搭建骨架（区域设计/事件链/世界脉络）、提炼方法论 | 填充血肉（NPC 小传/怪物图鉴/区域生态/历史事件） |
| **世界观后续** | 从叙事提炼机制、写 PRD、做技术方案 | 可担任翻译官：阅读叙事输出后提炼游戏机制草稿；可担任约束者：给下一批叙事设定"可转化性要求" |
| **性能** | 本地开发不受限；**线上运行受 2核2G 服务器约束**（见第 5.4 节） | 性能高，可代劳部分游戏计算（见第 5.4 节） |

**分工前提**：黑白机无法同时在线，不能做实时接力。但双方各自上线时都可担任「翻译官」或「约束者」——任何一方产出了叙事或机制内容，另一方上线后阅读、提炼、标注反馈，再推动下一轮产出。

**通用迭代流程**（双向适用，不限定白机做翻译、黑机做创作）：

```
【一方】产出骨架/叙事/机制草稿
       ↓
【另一方】阅读后提炼
          - 标注：✅ 可直接用 / ⚠️ 需调整 / ❌ 无法转化
          - 给出可转化性约束（下一批产出必须满足的要求）
       ↓
【原方】根据反馈修改 + 产出下一批
       ↓
    循环，直到沉淀为可执行的设定集
```

**具体分工建议**（根据双方上线时间灵活安排）：
- **黑机主导阶段**：黑机用酒馆搭建世界观骨架 → 白机上线后阅读 → 提炼游戏性草稿 → 标注反馈 → 黑机下次继续叙事
- **白机主导阶段**：白机写机制草稿 → 黑机上线后阅读 → 补充叙事细节 → 标注可行性 → 白机定稿 PRD
- **关键原则**：每一轮产出都附带「可转化性标注」，避免叙事跑偏到无法做成游戏

## 2. 战斗系统

### 2.1 V1 错误修正

V1 推荐"塔楼楼层副本"（战斗在塔楼二三层），这是**方向性错误**。被"塔楼已渲染闲置"的技术现状绑架，没有从游戏设计角度思考。

**修正方向**：
- 塔楼三层是**新手村/安全区**，全部开放给所有人，是共同的复活点（初入德塔在此复活）
- 战斗在**塔外**——走出塔楼大门，进入近塔森林及更远的塔外区域
- 塔楼 = 据点（社交/教学/复活），塔外 = 冒险区（战斗/打怪/探索）

### 2.2 区域定位修正

| 区域 | V1 错误定位 | V2 修正定位 |
|------|-----------|-----------|
| 塔楼一层大厅 | 聊天社交 | 社交 + 复活点 + 传送门 |
| 塔楼二层房间 | 战斗区（错误） | 教学/训练（新手学习操作） |
| 塔楼三层哨位 | BOSS 区（错误） | 安全区观景/预留 |
| 塔楼大门 | 彩蛋（错误） | **战斗入口**——走出去进入森林 |
| 近塔森林 | 不存在 | **新手战斗区**——封印裂隙、击杀低级怪物 |
| 裂缝外围 | 不存在 | 中级战斗区——精英怪物、更多裂隙 |

### 2.3 战斗模式设计

**参考泰拉瑞亚的轻量 ARPG 风格**：

- **鼠标控制朝向**：玩家精灵始终面朝鼠标光标方向。在 2D 侧视角中表现为左右翻转（鼠标在左→面向左，鼠标在右→面向右）
- **左键攻击**：点击鼠标左键 → 玩家朝鼠标方向发起攻击。近战武器=挥砍（扇形范围），远程武器=投射物（沿鼠标方向飞出）
- **怪物有自主攻击手段**：不是等玩家撞上来才打。怪物有自己的行为模式：
  - **近战型**（史莱姆类）：发现玩家后追击，碰撞造成伤害。部分怪物会"蓄力冲锋"（加速冲向玩家最后位置）
  - **远程型**（虚空射手类）：保持距离，朝玩家方向发射投射物（黏液弹/能量箭）
  - **范围型**（裂隙行者类）：玩家靠近时触发范围攻击（脚下出现预警圈 → 延迟爆炸）
  - **召唤型**（虚空母体类）：定期召唤小怪，本体脆弱但有小怪保护
- **战斗节奏**：单次战斗 < 30 秒。怪物攻击有明显的预警信号（闪烁/蓄力动画），给玩家反应时间。不拖沓，边打边在聊天框调侃
- **死亡惩罚**：极轻——掉回塔楼一层复活点（安全区），不掉落任何东西
- **多人合作**：多个玩家同时攻击同一怪物 → 伤害叠加。即兴合作，不需要组队系统。怪物血量按在场玩家人数微量缩放（3 人打 = 怪物血量 x1.2，不是 x3，保持单人也能打）
- **走位是核心**：远程怪物需要玩家躲投射物，范围型怪物需要看预警圈走位。不是站桩输出

**核心战斗循环**（从世界观提炼）：

```
从塔楼大门走出 → 进入近塔森林 → 发现封印脱落的裂隙 → 怪物从裂隙漏出
→ 鼠标瞄准 + 左键攻击（走位躲远程/范围攻击）→ 击杀怪物 → 怪物掉落粘液/骨骼等材料
→ 可选：封印裂隙（获得额外奖励）→ 材料存入共享仓库 → 回塔楼休息/聊天
```

**战斗体验设计原则**：

| 原则 | 说明 | 泰拉瑞亚对应 |
|------|------|-------------|
| 预警 > 反应 | 怪物攻击前有 0.3-0.5 秒预警，给玩家反应时间 | 红名闪烁→爆炸 |
| 简单但有深度 | 基础操作只有"移动+左键"，但不同怪物组合创造变化 | 同一条战斗逻辑应对不同敌人组合 |
| 信息透明 | 怪物类型一眼能辨（颜色/形状/大小暗示行为模式） | 史莱姆=近战，射手=远程，一目了然 |
| 合作涌现 | 没组队系统，但自然产生配合（有人拉怪有人输出） | 多人同屏即合作 |

## 3. 装备系统

### 3.1 V1 错误修正

V1 推荐"无属性皮肤（纯外观，零数值）"，这是**矫枉过正**。

**修正方向**（用户明确指示）：
- **有数值系统，但人物本身没有等级**——变强靠装备不靠升级
- **装备数值差异尽量小**——通过外观、机制做出差异化，而不是数值碾压
- 比如攻击力从 5 到 15（差异 3 倍），不是从 5 到 500（差异 100 倍）

### 3.2 装备数值设计原则

**核心原则：装备数值是"锦上添花"，不是"必需品"**。没有装备也能打怪，有装备只是更轻松。

| 设计维度 | 原则 | 示例 |
|---------|------|------|
| **数值范围** | 装备间数值差异小（约 2-3 倍），天花板低 | 攻击力 5~15，防御力 3~10 |
| **差异化来源** | 靠外观和机制，不靠数值 | 火属性武器攻击力 8，但对冰属性怪物伤害 +50% |
| **无等级限制** | 人物无等级，装备无穿戴等级要求 | 新人拿到顶级装备也能穿 |
| **不绑定数值成长** | 装备获取靠打怪掉落，不靠升级解锁 | 没有"30级才能穿"的限制 |

### 3.3 装备机制差异化（替代数值差异化）

既然数值差异小，就用**机制差异**制造装备的独特感：

| 装备 | 类型 | 数值 | 独特机制 | 来源 |
|------|------|:--:|---------|------|
| 木剑 | 近战 | ATK 5 | 无 | 新手默认 |
| 骨刃 | 近战 | ATK 7 | 对裂隙怪物 +20% 伤害，挥砍范围略大 | 普通怪物掉落骨骼合成 |
| 虚空裂片 | 近战 | ATK 8 | 攻击有概率撕裂空间（额外伤害） | 裂隙封印奖励 |
| 史莱姆弹弓 | 远程 | ATK 4 | 投射物弹跳 2 次，可绕障碍 | 史莱姆类怪物掉落黏液合成 |
| 骨弓 | 远程 | ATK 6 | 投射物穿透 1 个小怪 | 骨骼+裂隙材料合成 |
| 黏液护甲 | 防具 | DEF 5 | 受击时减速攻击者 | 史莱姆类怪物掉落 |
| 精英战利品 | 近战/远程 | ATK 10 | 自带 1 个成品附魔效果 | 精英怪物掉落（已掠夺过的） |

**关键**：每件装备有"自己的故事"——这把骨刃是用我们打过的怪物的骨头做的，那把虚空裂片是封印裂隙时获得的。装备不只是数值，是世界观的延伸。

### 3.4 装备获取途径（从世界观推导）

```
普通怪物 → 掉落粘液/骨骼/甲壳等材料 → 材料合成基础装备
                                    ↓
精英怪物 → 自带成品装备（因为已掠夺过了）→ 直接获取
                                    ↓
裂隙封印 → 获得虚空能量/裂片 → 合成高级装备
                                    ↓
区域 BOSS → 掉落独特材料 → 合成该区域专属装备
```

### 3.5 技术关键点

| 需要新增 | 可复用 |
|---------|--------|
| `Material`/`Equipment`/`Inventory` 数据库表 | Prisma 迁移模式 |
| 装备 API（穿戴/卸下/合成） | E 键交互框架 |
| 装备数值计算（攻击/防御/特殊效果） | skinId 机制（装备图层叠加） |
| 共享仓库（大厅可交互对象） | 现有家具摆放模式 |

## 4. 怪物系统

### 4.1 核心问题

20 人社区需要什么样的怪物？V1 的"每日世界 BOSS + 每周轮换"大方向对，但**位置错了**——BOSS 不在塔楼三层，而在塔外区域（森林/裂隙/腹地）。

### 4.2 怪物类型与分布

| 怪物类型 | 区域 | 刷新逻辑 | 难度 | 掉落 | 社交性 |
|---------|------|---------|:--:|------|:------:|
| **普通怪物** | 近塔森林 | 裂隙定时刷新，裂隙存在期间持续出怪 | 低 | 基础材料（粘液/骨骼/甲壳） | 单人可打 |
| **精英怪物** | 裂缝外围 | 裂隙刷新时小概率出精英 | 中 | 基础材料 + 成品装备（已掠夺） | 2-3 人合作 |
| **区域 BOSS** | 各区域深处 | 每日定时刷新（晚 20:00-22:00） | 高 | 独特材料 + 装备配方 + 全服公告 | 3-5 人合作 |
| **裂隙 BOSS** | 深渊裂隙 | 事件触发（全社区累积材料达标后出现） | 最高 | 独有装备配方 + 成就 | 全社区动员 |

### 4.3 裂隙机制（世界观驱动的核心玩法）

裂隙是怪物来源，也是**可交互对象**：

```
裂隙生命周期：
1. 裂隙在森林中随机出现（定时刷新，如每 30 分钟）
2. 裂隙持续泄漏怪物（普通怪物从裂隙中走出）
3. 玩家可以选择：
   a. 击杀怪物（获得材料）
   b. 封印裂隙（获得额外奖励，该区域暂时安全）
4. 裂隙一段时间后自行关闭（但如果玩家不封印，期间怪物会越来越多）
5. 裂隙关闭后，该区域的安全状态维持一段时间
```

**裂隙的游戏性意义**：
- **目标感**：玩家"看到问题（裂隙）→ 解决问题（封印）→ 获得奖励"
- **时间压力**：裂隙不封会越来越难，但不是硬性 deadline（没有惩罚，只是更麻烦）
- **社交触发**：发现大裂隙 → 群里喊人 → 一起封印 → 共同成就感

### 4.4 怪物设计（契合世界观调性）

怪物名称和形象由黑机世界观创作决定，但设计原则提前定好：

- **命名**：从虚空生物生态推导，如"虚空幼虫"、"裂隙行者"、"虚空猎手"
- **形象**：32×32（小怪）/ 64×64（精英/BOSS），像素风
- **四种攻击行为模式**（玩家需要针对不同模式调整走位）：
  | 模式 | 描述 | 怪物示例 | 玩家应对 |
  |------|------|---------|---------|
  | **追击型** | 发现玩家后直线追击，碰撞造成伤害 | 虚空幼虫、史莱姆 | 边打边退，注意别退到死角 |
  | **远程型** | 保持中距离，朝玩家方向发射投射物 | 虚空射手 | 走位躲弹幕，找掩体 |
  | **范围型** | 玩家靠近时脚下出现预警圈 → 延迟范围伤害 | 裂隙行者 | 看预警圈闪躲，远离圆心 |
  | **召唤型** | 定期召唤小怪，本体脆弱 | 虚空母体 | 优先清小怪，再打本体 |
- **AI 复杂度**：不做复杂寻路。追击型=朝玩家直线跑；远程型=保持距离+朝玩家射击；范围型=玩家进入范围→触发预警→延迟爆炸；召唤型=定时生成小怪。行为树只有 3-4 层
- **弱点**：由世界观设定推导（如"虚空生物怕光"→ 装备火属性武器有加成，远程型怕打断）
- **数量**：考虑线上服务器 2核2G 性能约束，同屏怪物不超过 15 个（见第 5.4 节性能方案）
- **预警原则**：所有怪物攻击前有 0.3-0.5 秒视觉信号（闪烁/变色/范围圈），玩家靠预警走位而非靠反应速度

### 4.5 技术关键点

| 需要新增 | 可复用 |
|---------|--------|
| `Monster`/`MonsterSpawn` 数据库表 | `PlayerState` schema 定义模式 |
| 怪物 AI 脚本（巡逻/追击/碰撞） | `Player.js` 移动逻辑（怪物 = AI 控制的玩家） |
| 裂隙对象（定时刷新/关闭/封印） | E 键交互框架 |
| 定时刷新服务（服务端） | `NetworkSystem` 同步 + `onStateChange` diff |
| BOSS 击杀全服公告 | `sendNpcReply` 广播模式 |

## 5. 游戏性（留存设计）

### 5.1 核心问题

为什么成员每天想来德塔？V1 的"三大留存支柱 + 轻量成长"大方向对，但需要基于世界观重新提炼。

### 5.2 三大留存支柱（世界观驱动版）

#### 支柱 1：每日仪式感

**从世界观提炼**：空间裂隙每天不定时出现 → 玩家每天都有"去查看裂隙"的理由。

| 机制 | 描述 | 世界观关联 |
|------|------|-----------|
| 裂隙巡逻 | 每天上线看一眼塔楼外的裂隙状态 | "裂隙难以完全封印，时不时就有漏进来" |
| 德塔签到 | 签到记录"我在德塔的第 X 天" | 渐进探索的时间标记 |
| 每日裂隙报告 | 男德通播报今日裂隙情况（出现在哪个区域、预计多久关闭） | 利用现有 NPC 广播系统 |

#### 支柱 2：共同目标

**从世界观提炼**：裂隙无法完全封印 → 但可以通过集体努力延缓 → 社区共同目标。

| 机制 | 描述 | 世界观关联 |
|------|------|-----------|
| 裂隙封印进度 | 全社区累积封印次数，达到阈值 → 解锁新区域 | "肃清周围森林"是男德学院的目标 |
| 塔楼建设 | 用怪物材料装饰/扩建塔楼 | 德塔是"落脚点"，越住越像家 |
| 共享装备库 | 全社区共享材料，一起合成装备 | 怪物材料用途 |

#### 支柱 3：人情记忆

**从世界观提炼**：每次封印裂隙、击杀 BOSS、解锁新区域，都是"我们一起做过的事"。

| 机制 | 描述 |
|------|------|
| 战绩墙 | 塔楼一层可交互对象，记录 BOSS 击杀/裂隙封印记录 |
| 战利品陈列 | 击杀精英怪物掉落的装备/材料，摆在大厅展示 |
| 区域探索记录 | "我们发现了裂缝外围"——首个到达该区域的成员留名 |
| 年度回顾 | 年末男德通生成"德塔年度报告" |

#### 支柱 4（可选）：轻量成长

- **装备驱动成长**：人物无等级，变强靠装备。装备数值差异小（2-3 倍），天花板低
- **区域解锁**：累计参与 BOSS/封印裂隙达到阈值 → 解锁新区域。这是"成长"的唯一体现
- **设计哲学**：成长是"我去过哪里的证明"，不是"我比谁强的证明"

### 5.3 游戏性提炼方法论

V2 的核心方法论——从世界观叙事中提炼游戏性，不是反过来：

```
世界观设定                   游戏性机制
─────────────────           ─────────────────
"空间裂隙时不时出现"    →    定时刷新机制（裂隙出现→怪物出没）
"怪物只有本能没有理智"  →    简单 AI（巡逻/追击/碰撞，不做复杂行为）
"裂隙无法完全封印"      →    目标感（封印=获得奖励，但不强制）
"装备是怪物材料做的"    →    收集系统（打怪→材料→合成）
"男德通有秘密"          →    对话探索（多次对话解锁隐藏剧情）
"原住民在抗争"          →    NPC 支线（野外遇到原住民 NPC）
"塔楼是落脚点"          →    安全区/社交中心/复活点
"深渊有隐藏区域"        →    事件解锁（全社区达标后开放）
```

### 5.4 黑机性能代劳方案

| 代劳内容 | 方案 | 理由 |
|---------|------|------|
| **怪物 AI 计算** | 黑机跑游戏服务器实例（已部署 WebSocket 架构），怪物 AI 在低延迟黑机上计算 | R-005 已有 WebSocket 外包先例，黑机 R7 9700X 性能远高于云端服务器 |
| **世界状态服务** | 黑机运行独立 World State 服务（裂隙定时刷新/怪物巡逻/区域状态），云端仅做客户端转发 | 降低云端 CPU 负担，怪物 AI 是主要瓶颈 |
| **AI NPC 对话** | 黑机本地运行 DeepSeek API 调用，NPC 对话不在云端推理 | 男德通已有 SSE 流式对话，黑机可分担推理压力 |
| **不做代劳** | 玩家位置同步/碰撞检测/聊天广播 | 必须低延迟（<50ms），云端服务器 2核2G 可承受 |

## 7. 不做的事

| 不做 | 理由 |
|------|------|
| ❌ PVP | 纯PVE，熟人圈 PK 容易产生矛盾 |
| ❌ 人物等级系统 | 变强靠装备不靠升级，避免数值碾压 |
| ❌ 经济系统（金币/拍卖行/交易） | 20人经济体量太小，共享仓库已够 |
| ❌ 复杂 AI（寻路/BOSS阶段切换/技能连锁） | 四种基础行为模式（追击/远程/范围/召唤）验证核心循环，不做复杂寻路和 BOSS 多阶段 |
| ❌ 排行榜/伤害统计 | 制造竞争焦虑，违背社交驱动 |
| ❌ 付费/内购/氪金 | 朋友圈限定社区，永不做商业化 |
| ❌ 主线叙事 | 用世界脉络替代主线，永远有东西探索 |

## 8. 分阶段实施建议
