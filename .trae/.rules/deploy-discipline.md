# 部署纪律

## 核心规则

1. **禁止**在用户未明确要求部署时，执行任何部署到测试环境或生产环境的操作
2. **明确要求**指用户说出以下关键词之一：
   - "部署到测试"
   - "部署到正式"
   - "部署到生产"
   - "上线"
   - "发布"
3. 开发完成后，只能：
   - 本地构建验证（`npm run build`、`tsc --noEmit`）
   - 本地运行测试
   - 提交代码（git commit + push）
4. **严禁**自行执行 `scp`、`rsync`、`systemctl restart` 等部署命令
5. 用户要求部署时，必须确认目标环境（测试 or 正式），确认后再执行

## 原因

部署是不可逆操作，未经确认的部署可能导致线上事故。所有部署必须由用户主动发起。
