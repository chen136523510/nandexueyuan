# Python 环境使用与开发规范

## 核心原则与环境管理 (uv 强制)
本项目全面拥抱 `uv` 作为统一的 Python 环境与包管理器。**绝对禁止直接使用原生的 `python` 或 `pip` 命令**。

### AI 终端执行纪律
在需要运行代码或安装依赖时，必须严格遵守以下指令，**无需尝试手动激活虚拟环境**：
- **初始化环境**：`uv venv`（首次运行前检测 `.venv/`，无则创建）
- **安装新依赖**：`uv pip install <pkg>`
- **执行脚本**：`uv run python <script>.py`（`uv` 会自动路由到本地 `.venv/`）

## 跨平台与路径规范
- **拒绝硬编码路径**：严禁使用字符串拼接路径。
- **强制使用 Pathlib**：所有文件读写、目录创建、路径解析操作，必须统一使用 Python 标准库 `pathlib.Path`。

## 代码风格与工程质量
- **极简依赖**：基础工具脚本尽量仅使用 Python 标准库，降低使用者环境初始化成本。
- **强制类型注解 (Type Hinting)**：所有函数签名必须包含明确的入参和出参类型注解。
- **防御性异常处理**：必须使用 `try...except` 捕获具体异常（如 `FileNotFoundError`），并输出友好的中文控制台提示，严禁抛出导致崩溃的裸露 Stacktrace。
