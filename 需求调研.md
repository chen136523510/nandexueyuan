## 5. 技术方案（已确定）

### 整体架构

```
用户浏览器
    ↓ HTTPS
Nginx（宝塔自带，反向代理 + 静态托管）
    ↓ /api/* 转发
Node.js 后端（端口 3000，PM2 守护）
    ↓
SQLite 数据库
```

### 技术栈

| 层 | 技术 | 理由 |
|---|------|------|
| **前端框架** | Vue 3 + Vite + TypeScript | 主流、构建快、类型安全 |
| **前端UI库** | Element Plus（待定） | 中文友好、组件丰富 |
| **前端路由** | Vue Router | Vue 官方 |
| **前端状态** | Pinia | Vue 3 官方推荐 |
| **HTTP 客户端** | Axios | 标配 |
| **后端框架** | Express（待定，Fastify也可选） | 教程多、生态成熟 |
| **数据库** | SQLite（推荐）| 20人使用，零运维，几乎不占内存 |
| **ORM** | Prisma | 类型安全、迁移方便 |
| **认证** | JWT + bcrypt | 标准方案 |
| **进程管理** | PM2 | 守护 Node 进程，开机自启 |
| **反向代理** | Nginx（宝塔自带） | 转发 + SSL证书 |

### 部署流程

1. 本地开发：Vue + Node 都在本地跑
2. Vue 本地构建：`npm run build` 生成静态文件
3. 上传到服务器：通过 git push + 服务器 git pull
4. 服务器上：
   - Vue 的 `dist/` 由 Nginx 直接托管
   - Node 后端通过 PM2 启动

### 内存预算（1.6 GiB 内可承受）

| 组件 | 占用 |
|------|------|
| 系统 + 宝塔 | ~400 MB |
| Nginx | ~50 MB |
| Node.js 后端 | ~80-150 MB |
| SQLite | ~10 MB |
| **总计** | **~600 MB**，剩余 ~1 GB 余量充足 |

### 已排除的方案

- **Next.js**：构建峰值需 1.5-3 GB 内存，1.6 GiB 服务器无法本地构建（需本地构建 + 上传镜像，部署复杂）
- **Java/Spring Boot**：内存吃太多
- **Python/Django**：可选但 Vue 已选 JS 栈，统一用 Node 更简单
- **PHP**：偏老，学习价值不如 Node.js

### 选购指南（已决策 + 说明）

#### 镜像选择：Ubuntu 22.04 LTS ✅

| 常见镜像 | 特点 | 适合 |
|----------|------|------|
| **Ubuntu 22.04 LTS** | 社区大、教程多、软件包丰富 | ✅ 推荐，新手首选 |
| CentOS 7 | 稳定，但已停止维护（2024年EOL） | ❌ 不推荐 |
| CentOS Stream | CentOS替代品，滚动更新 | 企业用户，新手不推荐 |
| Debian 12 | 极度稳定，比Ubuntu更轻量 | 有Linux经验的用户 |
| Alibaba Cloud Linux 3 | 阿里云定制，兼容CentOS | 阿里云深度用户 |

选Ubuntu 22.04 LTS的理由：LTS=长期支持（支持到2027年），网上教程最多，遇到问题最容易搜到答案。

#### 拓展程序选择：只选宝塔面板免费版 ✅

| 拓展程序 | 是否选择 | 理由 |
|----------|----------|------|
| **宝塔面板免费版** | ✅ 选 | 可视化管理服务器，新手友好，装完后可在网页上管理一切 |
| Java | ❌ 不选 | 除非用Java开发后端，否则浪费资源 |
| Python | ❌ 不选 | 可以后续在宝塔面板中按需安装 |
| Linux桌面环境 | ❌ 不选 | 极度浪费服务器资源（2G内存扛不住），服务器不需要桌面 |

2GiB内存很小，尽量少装东西。宝塔面板本身占约300-500MB内存。

#### 存储云盘选择

| 选项 | 建议 | 说明 |
|------|------|------|
| ESSD Entry | ✅ 推荐 | 便宜够用，个人站足够 |
| ESSD PL0 | 性能更好 | 贵一些，20人访问不需要 |
| ESSD PL1/2/3 | 高性能 | 企业级，完全不需要 |

容量建议：40GB起步（系统+宝塔+网站足够），不够后续可扩容。

#### 带宽计费模式与带宽值

| 计费模式 | 说明 | 建议 |
|----------|------|------|
| **按固定带宽** | 买多少用多少，超了不扣费 | ✅ 推荐，费用可控 |
| 按使用流量 | 用多少算多少，流量大时费用不可控 | 不推荐新手 |

带宽值是什么：带宽值=最大传输速度。1Mbps ≈ 128KB/s。

| 带宽值 | 速度 | 适合场景 |
|--------|------|----------|
| 1 Mbps | ~128KB/s | 纯文字站勉强够 |
| **3 Mbps** | ~384KB/s | ✅ 推荐，图文站够用 |
| 5 Mbps | ~640KB/s | 有图片较多时更流畅 |

20人访问，3Mbps基本够用，不够后续可升。

#### IPv4端口/协议

| 端口 | 是否勾选 | 理由 |
|------|----------|------|
| SSH (22) | ✅ 必须 | 远程连接服务器 |
| RDP (3389) | ❌ 不需要 | Windows远程桌面，你选了Ubuntu用不上 |
| **HTTP (80)** | ✅ 必须 | 网站访问（备案前调试用） |
| **HTTPS (443)** | ✅ 必须 | 加密访问（上线后必备） |

**HTTP和HTTPS都必须勾选**，否则网站无法通过浏览器访问。
